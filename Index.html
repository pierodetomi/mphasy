Bring MVVM to PHP


mphasy is a PHP framework for developing MVVM web applications.
It's easy, lightweight, and based on a Convention over configuration approach.

Templated Views & View-Model Data

With templated views you don't need to write every time an entire html page from scratch, and you won't even need to copy-paste your theme markup every time for each new page.
Finally you'll be able to bind data from view-model in your views: just point to $this->ViewModel->* and you're done!

<html>

      @PartialView('footer','Account','GetFooter')
   </body>

Templates & Partial Views

As easy as you see just above! mphasy allows the implementation of "templates", to be used as back-bone structure for multiple pages across your application.
Then, you render partial views and the main view inside it, calling specific framework methods, such as @RenderPartialView() and @MainView().

Almost no config

You know.
Configuration is always THE boring part. What if we could configure almost nothing, just copy-paste library files in one folder, tweak a line of code or two, and run in the browser?
Well, this is one of the main goals of mphasy: convention over configuration is the key concept.

Architecture

mphasy is not about UI, it's about architecture.
Implementing the MVVM design pattern, it allows a clear separation between presentation logics (views) and application logics (view-models).
If you're a precise software developer and want to write clean application/presentation code, mphasy is the right choice to go.

3rd Party Libraries

Though mphasy has been developed free from any dependency with any 3rd party library, it remains their friend.
In fact, being it an architectural framework, you might still need to use some library to prettify your UI: examples could be Twitter Bootstrap or jQueryUI.
But if you prefer, you still can define all of your styles by hand: it's really up to you.
Plus, if you want to enable client-side data‐binding in your views, you could use the awesome library KnockoutJS.
